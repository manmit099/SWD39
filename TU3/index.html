Concept explanations

HTML Structure
Every HTML document follows a fundamental structure containing the <!DOCTYPE html> declaration, followed by the root <html> tag which encompasses the <head> and <body> sections. These elements formulate the skeleton of any website.
HTML Tags: Pair vs Self-Closing
There are two main types of tags in HTML. Pair Tags have an opening and closing tag (e.g., <body>...</body>). Self-Closing Tags don't require a closing pair and end within themselves (e.g., <meta>).
The Head Section
The <head> is essentially the brain of the webpage. It contains metadata, the page <title>, linked CSS stylesheets, and scripts that shouldn't be rendered as visible text on the actual page.
SEO & Meta Tags
Meta tags store data about the document (Metadata). Important attributes like name="description" inform Search Engines (like Google) what the page is about. Consistently setting an accurate title and description is a foundational SEO practice.
Attributes
Attributes are key-value pairs placed inside tags to provide additional information about that tag. For example, in <html lang="en">, lang is the attribute and en is the value indicating the document is in English.

Syntax explained

SYNTAXDESCRIPTION
<!DOCTYPE html>Declares the document as an HTML5 document.
<html>The root element that wraps all content on the entire page.
<head>Container for metadata, title, scripts, and stylesheets. Not visible on the page.
<body>Defines the document's body. Contains all visible content (text, images, links).
<title>Sets the title shown in the browser tab and used by search engines.
<meta>Self-closing tag used to provide metadata like charset, description, and viewport settings.
<link>Used to link external resources, typically CSS stylesheets.
<script>Used to embed or link executable JavaScript code.
[04]

Tips & Warnings

Copy-pasting boilerplate code (like basic HTML structures or forms) from trusted sites like MDN or codewithharry.com is perfectly fine. Focus your brainpower on the logic that makes your app unique.
Changing the <title> and adding a descriptive <meta name="description"> is an excellent SEO practice.
Hover over attributes in VS Code (like name="description") to read built-in documentation about what they do.
When linking an HTML file with CSS/JS, if changes in CSS don't show up immediately in Live Preview, you might occasionally need to trigger a manual reload.
To preview your website on a mobile device, your PC's Wi-Fi network must be set to Private. It won't work on a Public network profile.
[05]

Step-by‑step walkthrough: Mobile Preview

01
Open Terminal
Right-click in VS Code and open the terminal (or use Command Prompt / PowerShell).
02
Find your IP Address
Type ipconfig and hit Enter. Look for the IPv4 address (e.g., 192.168.1.199) and copy it.
03
Configure Live Preview
Open VS Code Settings, search for "Live Preview". Find the setting "Local IP host address to host your files on" and paste your copied IPv4 address.
04
Set Network to Private
Go to your computer's Wi-Fi settings and ensure your network is set to "Private Network" (not Public).
05
Restart and Preview
Restart VS Code. Right-click your HTML file and select "Show Preview".
06
Open on Mobile
Take your mobile phone (connected to the same Wi-Fi) and type your IP address and port into the browser (e.g., 192.168.1.199:5000 or 3000) to view the live site.
[06]

Quick summary

An HTML file generally follows a strict boilerplate structure: <!DOCTYPE>, <html>, <head>, and <body>.
Tags can be Pair Tags (with open/close counterparts) or Self-Closing Tags (like <meta>).
The <head> tag holds metadata, links to CSS, and the page title. It's crucial for SEO.
The <body> tag contains everything visually presented on the web page.
Attributes inside tags (like lang="en") define extra behavior or characteristics of the tag.
Copy-pasting standard boilerplate code is a normal and efficient practice for developers.
You can preview your local VS Code environment on your smartphone by routing Live Preview through your IPv4 address over a Private Network.
